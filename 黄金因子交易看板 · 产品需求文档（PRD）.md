## 1. 背景与目标

### 1.1 产品背景

黄金价格受美元走势、实际利率、通胀预期、地缘政治、央行购金等九大宏观维度共同驱动，且各因子在不同市场状态（Regime）下的有效性截然不同。传统交易者依赖主观判断割裂地观察这些指标，缺乏系统化的因子量化框架。

本产品旨在将九大黄金驱动因子整合为一个**可量化、可追踪、可执行**的智能交易看板，并通过 VSTAR.COM 平台的仿真模式，让用户在零风险环境下验证策略有效性后，再决定是否转入实盘。

### 1.2 用户画像

|用户类型|描述|核心诉求|
|---|---|---|
|量化自主交易者|有一定量化基础，希望系统化分析黄金因子|因子 IC 追踪、SHAP 归因可视化、信号自动生成|
|宏观主观交易者|依赖宏观分析但缺乏量化工具|直观的因子仪表盘、宏观矩阵总结、简洁信号提示|
|策略研究员|构建和验证黄金量化策略|完整因子工程流程、样本内外对比、仿真回测验证|

### 1.3 核心价值主张

- 将九大黄金驱动因子从「主观感知」转化为「可量化打分」
- 通过 SHAP 归因实时解释每日信号的驱动因子，消除黑箱
- 集成基于 ATR 的动态仓位管理，防止主观冲动干扰执行
- VSTAR 仿真模式提供零成本验证，降低策略上线风险
- Regime 识别系统自动切换因子权重，解决因子 Regime 依赖性问题

### 1.4 成功指标（KPI）

|指标|目标值|衡量方式|
|---|---|---|
|仿真账户样本外 Sharpe|> 1.0（3个月观察期）|VSTAR 仿真账户净值曲线|
|因子 IC 覆盖率|九大因子均完成历史 IC 计算|IC 时序图完整性检查|
|SHAP 归因响应时间|每日信号 < 3 秒生成完毕|前端性能监控|
|最大回撤（仿真）|< 15%|动态回撤计算模块|
|仓位管理规则执行率|100%（系统强制，不可人工绕过）|风控日志审计|

## 2. 产品架构总览

### 2.1 三大模块结构

|模块|子系统|职责|
|---|---|---|
|**模块 A：因子分析引擎**|数据采集 / 因子计算 / IC 分析 / 正交化 / Regime 识别|将原始宏观数据转化为标准化因子值，持续追踪有效性|
|**模块 B：交易信号引擎**|XGBoost 推理 / SHAP 归因 / 信号生成 / 仓位计算|基于因子输出做多/做空/中性信号，并计算精确仓位|
|**模块 C：仿真交易执行**|VSTAR API 对接 / 下单管理 / 止损追踪 / 账户监控|将信号与仓位指令提交至 VSTAR 仿真账户，追踪执行结果|

### 2.2 数据流架构

```
① 数据采集层
   FRED / Bloomberg / BofA / WGC / VSTAR 行情
          │
          ▼
② 因子计算层
   原始数据 → Z-Score 标准化 → Löwdin 正交化 → 九因子矩阵
          │
          ▼
③ 模型推理层
   正交因子矩阵 → XGBoost → 预测值 → SHAP 分解
          │
          ▼
④ 仓位计算层
   预测方向 + ATR + 账户状态 → 精确仓位数量
          │
          ▼
⑤ 执行层
   仓位指令 → VSTAR API → 仿真成交 → 实时持仓更新
          │
          ▼
⑥ 监控层
   持仓状态 → 止损检查 / 回撤计算 / 因子健康预警
```

### 2.3 技术栈建议

|层级|推荐技术|说明|
|---|---|---|
|后端计算|Python（pandas / numpy / xgboost / shap）|因子工程与模型推理核心|
|数据调度|Apache Airflow 或 Celery Beat|定时数据采集与因子更新|
|数据存储|TimescaleDB（时序）+ PostgreSQL（配置）|支持高效时序查询|
|前端看板|React + TailwindCSS + Recharts / D3.js|响应式看板，支持暗色模式|
|VSTAR 对接|VSTAR REST API / WebSocket|实时报价接收 + 仿真下单|
|告警推送|Telegram Bot / 邮件 / 企业微信|触发式推送，不打扰日常操作|

## 3. 因子分析看板（模块 A）

### 3.1 九大因子定义与数据源

每个因子均须满足：**有明确经济假说**、使用 `available_date`（实际公布日）对齐、252 日滚动 Z-Score 标准化。

|编号|因子名称|代理数据源|更新频率|预期 IC 方向|
|---|---|---|---|---|
|F1|美元强弱|DXY 指数 20 日变化率 Z-Score|日频|负相关 ↓|
|F2|货币政策预期|联邦基金期货 6M 隐含利率变化|日频|负相关 ↓|
|F3|**实际利率**（最强因子）|TIPS 10Y 实际收益率|日频|**强负相关 ↓↓**|
|F4|通胀预期|5Y5Y 通胀互换 / 盈亏平衡通胀率|日频|正相关 ↑|
|F5|地缘政治风险|GPR 指数（月频，日内 OVX 代理）|月/日代理|正相关 ↑|
|F6|市场情绪|GVZ + GLD Put/Call 比率合成|日频|正相关 ↑|
|F7|央行需求|WGC 央行净买入量（季频前向填充）|季频|正相关 ↑|
|F8|ETF 资金流|GLD + IAU 合计持仓量 5 日变化率|日频|正相关 ↑|
|F9|金矿产能|GDX vs 黄金相对强弱（产量代理）|日频|负相关 ↓|

> ⚠️ **数据对齐强制规范**
> 
> 所有宏观数据必须记录两个时间字段：
> 
> - `as_of_date`：数据报告期（如「3月31日」）
> - `available_date`：实际公布日期（如「4月28日」）
> 
> 看板的所有查询与计算必须使用 `available_date` 过滤，**严禁使用 `as_of_date`**。 这是防止前视偏差的工程生命线，不得以任何理由绕过。

### 3.2 因子看板主界面规范

#### 3.2.1 核心指标仪表盘（顶部通栏）

参考日报「核心指标仪表盘」设计，每张因子卡片展示：

- 因子名称 + 标识图标
- 当日 Z-Score 值（主数字，大字体）
- 原始数据值（次级展示，灰色小字）
- 日变化方向箭头（↑/↓）+ 变化幅度
- 52 周百分位色块（绿/黄/红渐变）
- 一句话信号解读（如「实际利率极高，看空压力」）

#### 3.2.2 因子 IC 追踪面板

每个因子独立展示滚动 252 日 IC 时序折线图，包含：

- 当前 20 日 IC 均值（数字标注）
- ICIR = 均值 / 标准差（达标线 0.5 红色虚线）
- IC 阈值参考线（±0.05 绿色虚线）
- Regime 着色背景（加息期 / 降息期 / 震荡期 用不同底色区分）

#### 3.2.3 Regime 稳定性热力图

- **横轴**：时间（按月）
- **纵轴**：九个因子
- **颜色深浅**：SHAP 贡献度大小
- **颜色冷暖**：方向（暖色 = 看多贡献，冷色 = 看空贡献）

这是识别因子 Regime 切换最直观的工具。

#### 3.2.4 因子相关性矩阵

实时展示九因子之间的 60 日滚动相关性矩阵热图。当任意两因子相关性超过 **0.7** 时，对应格子高亮红色警告——提示正交化效果可能需要更新。

### 3.3 Löwdin 正交化规范

- 使用 252 日滚动窗口，每个交易日重新计算协方差矩阵
- 正交化结果存入独立数据表，与原始因子值并行保存
- 界面提供「原始值 / 正交化值」切换开关，方便因子调试
- 当协方差矩阵条件数超过 1000 时，触发「正交化质量告警」

## 4. 交易信号引擎（模块 B）

### 4.1 模型规格

|参数|规格|设定理由|
|---|---|---|
|算法|XGBoost Regressor（预测 20 日收益率）|捕捉因子间非线性交互，适合宏观数据|
|训练集|2015-01-01 至 2022-12-31|覆盖加息、降息、QE、危机等多种 Regime|
|测试集|2023-01-01 至今（滚动更新）|真实样本外验证，不参与任何参数调优|
|预测目标|未来 20 日 XAU/USD 收益率|匹配慢变宏观因子的预测窗口|
|`max_depth`|4|黄金宏观样本有限，浅树防过拟合|
|`min_child_weight`|10|宏观事件稀少，防止在极端事件上过拟合|
|`learning_rate`|0.03|保守学习率，防止单棵树过度拟合|
|交叉验证|Purged CV（Walk-forward + Purge 20天 + Embargo 20天）|严防时序泄露|

### 4.2 信号生成逻辑

每日 UTC 21:00（美东收盘后 1 小时）运行完整推理，输出三档信号：

|信号|触发条件|含义|建议操作|
|---|---|---|---|
|🟢 **做多（Strong Buy）**|预测 20 日收益率 > +0.8%|九因子综合看多，方向明确|按仓位计算公式建立多头仓位|
|🟡 **中性（Neutral）**|预测收益率在 -0.8% ~ +0.8%|因子信号相互抵消，方向不明|不开新仓，持有现有仓位观察|
|🔴 **做空（Strong Sell）**|预测 20 日收益率 < -0.8%|九因子综合看空，方向明确|按仓位计算公式建立空头仓位|

### 4.3 SHAP 归因展示规范

#### 4.3.1 每日信号归因瀑布图

每次信号生成后，自动渲染 SHAP Waterfall 图：

- **基准值** E[f(x)]：标注历史均值预测
- **每个因子的 SHAP 值**：绿色条（正贡献/看多）、红色条（负贡献/看空）
- 条形宽度对应 SHAP 值绝对大小
- 鼠标悬停显示：因子原始值、Z-Score、SHAP 值、因子经济学解释

**示例归因输出：**

```
今日黄金预测：+0.72%（🟢 做多）

基准值 E[f(x)]            0.00%
F3 实际利率（↓2bps）   ── +0.28%  ← 最大贡献
F4 通胀预期（↑）       ── +0.19%
F6 市场情绪（GVZ↑）    ── +0.15%
F7 央行需求（持续）     ── +0.12%
F1 美元（DXY持平）      ── +0.02%
F5 地缘（GPR平稳）      ── -0.04%
F8 ETF资金流（流出）    ── -0.08%
F2 货币政策（中性）     ── +0.01%
F9 金矿产能（平稳）     ── +0.07%
                         ─────────
预测值                    +0.72%
```

#### 4.3.2 SHAP 异常检测规则

> ⚠️ **三类 SHAP 异常自动触发告警**
> 
> **异常 1**：单因子贡献度 > 50% → 模型过度依赖单一信号，暂停当日自动信号生成，等待人工确认
> 
> **异常 2**：某因子连续 30 日平均 SHAP 贡献下降 > 60% → 因子衰退预警，建议降低该因子在模型中的权重
> 
> **异常 3**：SHAP 方向与 IC 预期方向相反（如 F3 实际利率 SHAP 值为正） → Regime 切换预警，检查当前市场是否已进入非传统状态

### 4.4 Regime 识别系统

系统维护四种 Regime 状态，每日根据市场条件自动判定：

|Regime|识别条件|主导因子|特殊处理|
|---|---|---|---|
|**R1：实际利率主导**|F3 SHAP 贡献连续 20 日 > 35%|F3（最强压制/支撑）|F3 权重 ×1.5，F5/F6 权重 ×0.7|
|**R2：避险情绪主导**|GVZ > 40 或 VIX > 35 且 GPR 急升|F5 + F6 协同驱动|F5/F6 权重 ×1.5，注意强制平仓风险|
|**R3：央行需求主导**|WGC 买入量同比 > +30% 且 F7 SHAP > 25%|F7（与利率脱钩）|降低 F3 权重，F7 预测窗口延长至 60 日|
|**R4：通胀主导**|BEI > 3% 且 F4 SHAP > 30%|F4（抗缩水需求）|F4 权重 ×1.4，关注 F3 是否转负|

## 5. 仓位管理系统（模块 B · 子系统）

> 本模块严格实现《仓位管理核心指南》（Practical Framework for Risk-Adjusted Position Sizing）中的全套规则。**所有规则由系统强制执行，不提供人工覆盖选项。**

### 5.1 基础仓位计算公式

```
仓位数量（手数）= （账户总额 × 单笔风险比例）
                 ÷ （ATR × 止损倍数 × 每点价值）
```

**默认参数：**

|参数|默认值|说明|
|---|---|---|
|单笔风险比例|1.0%（健康档）/ 0.5%（预警档）/ 0%（熔断档）|随风险乘数自动调整|
|ATR 周期|14 日|衡量近期市场呼吸节奏|
|止损倍数|2.5×（黄金 XAU/USD 默认）|可调整为 2×~3×|
|每点价值|根据 VSTAR 合约规格自动获取|—|

**示例（仿真账户 $100,000）：**

```
风险金额  = $100,000 × 1% = $1,000
ATR       = $25（14日ATR）
止损距离  = $25 × 2.5 = $62.5
仓位数量  = $1,000 ÷ $62.5 = 16 手
```

### 5.2 动态风险乘数（三档分级）

|状态|触发条件|风险乘数|单笔风险|允许操作|
|---|---|---|---|---|
|🟢 **健康**|当前回撤 < 5%|1.0×|1.0%|正常建仓，所有信号均可执行|
|🟡 **预警**|当前回撤 5% ~ 15%|0.5×|0.5%|仅执行强信号（预测收益 > ±1.2%），减半仓位|
|🔴 **熔断**|当前回撤 > 15%|0.0×|0%|停止所有新仓，平仓现有持仓，进入复盘模式|

### 5.3 止损规则体系

#### 5.3.1 初始止损

|方向|止损位计算|
|---|---|
|做多|`止损位 = 建仓价 - 2 × ATR`|
|做空|`止损位 = 建仓价 + 2 × ATR`|

止损价格在建仓时即通过 VSTAR API 挂单，**不可人工删除**。

#### 5.3.2 保本止损（盈利保护）

- 当持仓浮盈达到 **1 × ATR** 时，系统自动将止损移至开仓成本价（保本线）
- 保本止损触发后，该笔交易最大亏损锁定为 **0**，同时释放风险预算额度

#### 5.3.3 吊灯移动止损（Chandelier Exit）

```
做多止损位 = max(建仓以来最高价) - 2.5 × ATR
做空止损位 = min(建仓以来最低价) + 2.5 × ATR
```

- 每日收盘后自动更新，通过 VSTAR API 修改挂单
- 止损位**只能向有利方向移动**，不可反向调整

### 5.4 Portfolio Heat 管理

**规则：全账户任意时刻的总风险暴露（Portfolio Heat）不得超过 2%。**

|方案|名称|操作逻辑|适用场景|
|---|---|---|---|
|**方案 A**|等比例缩减（Proportional Scaling）|所有持仓按同一比例压缩，确保总 Heat 回到 2% 以内|无法判断各仓位优先级时|
|**方案 B**|确定性排序（Priority Sizing）|按信号强度排序，优先保障高评分仓位的完整风险预算，剩余额度依次分配|各仓位信号强度有明显差异时|

> ⚠️ 系统每 **10 分钟**计算一次实时 Portfolio Heat。超过 **1.8%** 时触发黄色预警。

### 5.5 阶梯式恢复流程

账户进入预警/熔断状态后，必须经过系统性验证才能恢复：

|恢复阶段|进入条件|风险档位|晋升条件|退防条件|
|---|---|---|---|---|
|**观察期**|回撤超过 10% 触发|0.5%|连续 3 笔盈利，每笔触达移动止损锁利点|单次亏损 → 重置计数|
|**第一恢复档**|完成观察期验证|0.75%|最近 5 笔胜率 > 60% 且净值回升|**连续 2 次亏损 → 退回观察期**|
|**完全恢复**|完成第一恢复档验证|1.0%|净值回到回撤前最高点|回撤再次超过 5% → 重新触发预警|

> 💡 **双倍退防逻辑**：在升档过程中，连续出现 **2 次**亏损才强制退档（而非 1 次），原因是单次亏损可能只是市场噪音；连续两次才说明当前 Regime 未真正好转，升档尝试是错误的。

## 6. VSTAR 仿真交易对接（模块 C）

### 6.1 平台对接概述

VSTAR.COM 是一个提供多资产 CFD 交易的在线平台，支持黄金（XAU/USD）现货交易和仿真账户模式。系统通过 VSTAR 官方 API 实现数据获取与交易执行的自动化集成。

> 📋 **VSTAR 对接前置确认事项**
> 
> 1. 确认 VSTAR 提供 REST API 或 FIX 协议接口用于程序化下单
> 2. 申请仿真账户（Demo Account），获取 API Key 和账户 ID
> 3. 确认 XAU/USD 合约规格：最小手数、每手价值、点差、隔夜费率
> 4. 确认 API 频率限制（Rate Limit）以合理设计请求策略
> 5. 测试 WebSocket 连接稳定性，用于实时价格订阅和止损触发监控
> 
> 注：若 VSTAR API 不支持程序化止损修改，则在系统侧每 5 分钟检查一次并手动推送通知。

### 6.2 数据接入规范

|数据类型|接入方式|更新频率|用途|
|---|---|---|---|
|XAU/USD 实时报价|WebSocket 订阅|Tick 级别|ATR 计算、止损触发监控、实时盈亏|
|历史 K 线数据|REST API（OHLCV）|每日收盘后|ATR 计算（14日）、信号回测验证|
|仿真账户余额|REST API|每 10 分钟|Portfolio Heat 计算、风险乘数判定|
|仿真持仓明细|REST API|实时|止损追踪、保本止损触发判断|
|历史成交记录|REST API|每日|绩效统计、Sharpe 计算、回撤追踪|

### 6.3 下单流程规范

#### 6.3.1 开仓流程

1. 每日 UTC 21:30（美东收盘后 1 小时），系统完成当日信号生成
2. 信号为做多/做空时，调用仓位计算模块，输出精确手数
3. 检查 Portfolio Heat：若当前 Heat + 新仓位 Heat > 2%，触发等比例缩减或确定性排序
4. 通过 VSTAR API 提交 Market Order（开仓），同时挂载初始止损单
5. 开仓确认后，记录建仓价、手数、初始止损价至本地数据库

#### 6.3.2 止损更新流程

1. 每日收盘后，重新计算吊灯止损位（基于当日最高/最低价 + ATR）
2. 若新止损位优于当前止损位（有利方向），通过 API 修改挂单
3. 若浮盈首次达到 1×ATR，触发保本止损，将止损位移至成本价
4. 日内每 10 分钟检查一次实时价格，若触及止损位立即平仓（市价单）

#### 6.3.3 平仓条件

|平仓类型|触发条件|
|---|---|
|主动平仓|模型信号从做多/做空切换至中性或反向|
|止损平仓|价格触及吊灯止损位（系统自动执行）|
|熔断平仓|账户回撤超过 15%，强制平所有仓位|
|到期平仓|持仓满 40 个交易日（2 倍预测窗口），如未触及止损则强制平仓|

### 6.4 仿真账户仪表盘

实时展示内容：

- 账户净值曲线（日频，与 GLD 基准对比）
- 当前 Portfolio Heat 仪表盘（0-2% 进度条，1.8% 黄色、2% 红色）
- 当前风险乘数档位（🟢健康 / 🟡预警 / 🔴熔断）
- 持仓明细表：标的、方向、建仓价、当前价、浮盈、止损位、SHAP 主因
- 历史交易记录：每笔进出场原因、SHAP 驱动因子、盈亏
- 绩效指标：Sharpe（滚动60日）、Calmar、最大回撤、胜率、盈亏比

### 9.1 性能要求

|指标|要求|测量方式|
|---|---|---|

|ATR 仓位计算响应|< 1 秒（用户输入后）|前端交互计时|
|系统可用性|> 99.5%（每月）|Uptime 监控服务|

### 9.2 数据质量标准

- 每个因子每日缺失率 < 5%；超过此阈值当日信号标注「数据不完整」
- 所有宏观数据来源记录原始 API 响应，支持追溯验证

- VSTAR API Key 使用环境变量存储，**禁止硬编码或提交至代码库**
- 所有 API 调用使用 HTTPS，WebSocket 使用 WSS
- 仿真账户与实盘账户使用不同的 API Key，系统侧强制区分，防止误操作
- 交易日志不可删除，保留至少 2 年
